## Overview

A parameterized **Synchronous First-In First-Out (FIFO)** buffer with configurable data width and depth. Both read and write operations share the same clock domain. Includes full/empty status flags and protection against overflow (writing when full) and underflow (reading when empty).

FIFOs are fundamental building blocks in digital systems — used for data buffering between modules, rate-matching between producer and consumer, and pipeline decoupling.

## Architecture

```
         WRITE side                        READ side
  ┌──────────────────┐             ┌──────────────────┐
  │  wr_en           │             │  rd_en           │
  │  wr_data[W-1:0]  │──► MEMORY ──►  rd_data[W-1:0] │
  │                  │   ARRAY     │                  │
  └──────────────────┘  (D × W)   └──────────────────┘
         ▲                                  ▲
         │         ┌──────────┐             │
         └─────────│  WR_PTR  │─────────────┘
                   │  RD_PTR  │
                   │  COUNTER │──► full, empty flags
                   └──────────┘
```

## Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `DATA_WIDTH` | 8 | Width of each data word in bits |
| `DEPTH` | 16 | Number of entries the FIFO can hold |
| `ADDR_WIDTH` | 4 | log₂(DEPTH) — set automatically |

---

## I/O Ports

| Port | Direction | Width | Description |
|------|-----------|-------|-------------|
| `clk` | Input | 1 | System clock (all operations synchronous) |
| `rst_n` | Input | 1 | Active-low synchronous reset |
| `wr_en` | Input | 1 | Write enable — push data when HIGH and not full |
| `wr_data` | Input | DATA_WIDTH | Data word to write |
| `rd_en` | Input | 1 | Read enable — pop data when HIGH and not empty |
| `rd_data` | Output | DATA_WIDTH | Data word read from FIFO |
| `full` | Output | 1 | HIGH when FIFO is full — writes blocked |
| `empty` | Output | 1 | HIGH when FIFO is empty — reads blocked |
| `count` | Output | ADDR_WIDTH+1 | Current number of entries in the FIFO |

---

## Operating Conditions

| Condition | Behaviour |
|-----------|-----------|
| Write when not full | Data stored; write pointer incremented |
| Write when full | Write ignored; `full` flag asserted |
| Read when not empty | Data output; read pointer incremented |
| Read when empty | Read ignored; `empty` flag asserted |
| Simultaneous read + write | Both proceed; count stays the same |
| Reset | Pointers and count cleared; `empty` asserted |

## Concepts Covered

| Concept | Details |
|---------|---------|
| Circular Buffer | Read and write pointers wrap around using modulo indexing |
| Pointer Management | Separate read/write pointers with rollover logic |
| Full/Empty Flags | Generated by comparing pointer values and count register |
| Overflow Protection | Ignores write when full; prevents data corruption |
| Underflow Protection | Ignores read when empty; prevents X propagation |
| Parameterized RTL | Width and depth configurable at instantiation |

## How to Simulate

**Vivado:**
1. Add `src/sync_fifo.sv` as design source
2. Add `tb/tb_sync_fifo.sv` as simulation source
3. Run Behavioral Simulation
4. Verify `full`, `empty`, and `count` behave correctly across all test cases

**EDA Playground:**
1. Paste design and testbench, select SystemVerilog 2012
2. Run and check console output for PASS/FAIL messages

---

## Test Cases in Testbench

- **Fill test** — write DEPTH entries, verify `full` asserts, verify overflow protection
- **Drain test** — read all entries, verify `full` deasserts, verify `empty` asserts
- **Simultaneous R/W** — concurrent read and write, verify count stays constant
- **Reset test** — assert reset mid-operation, verify clean state recovery
- **Random test** — randomized read/write sequences verify pointer integrity

---

## Simulation Results

Waveforms and results are stored in `sim/`. Key signals to observe:
- `wr_ptr` and `rd_ptr` advance correctly
- `full` asserts exactly when count == DEPTH
- `empty` asserts exactly when count == 0
- `rd_data` correctly outputs stored values in FIFO order
